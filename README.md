# Quote Server & Client

Учебный проект: потоковая передача биржевых котировок через TCP + UDP.

Сервер генерирует случайные котировки и стримит их клиентам по UDP.
Клиент подписывается на нужные тикеры и выводит их в консоль.

## Структура проекта

```
rust-project-2/
├── common/          — общие типы (StockQuote), константы протокола
├── quote_server/    — сервер котировок
└── quote_client/    — клиент котировок
```


## Запуск

### 1. Подготовить файл тикеров

Создать файл `tickers.txt` — по одному тикеру на строку:

```
AAPL
GOOGL
TSLA
```

Файл используется и сервером (список тикеров для генерации),
и клиентом (список тикеров для подписки).

### 2. Запустить сервер

```bash
cargo run -p quote_server
```

Сервер запустится на TCP-порту `7777` и будет ждать подключений.

С подробными логами:

```bash
RUST_LOG=info cargo run -p quote_server
```

### 3. Запустить клиент (в отдельном терминале)

```bash
cargo run -p quote_client
```

С явными аргументами:

```bash
cargo run -p quote_client -- \
  --server-addr 127.0.0.1 \
  --udp-port 34254 \
  --tickers-file tickers.txt
```

Справка по аргументам:

```bash
cargo run -p quote_client -- --help
```

### 4. Остановить клиент

Нажать `Ctrl+C`. Клиент завершится корректно.
Сервер автоматически прекратит стриминг для этого клиента через 5 секунд.

## Протокол

### Инициализация (TCP)

Клиент подключается к серверу по TCP и отправляет команду:

```
STREAM udp://127.0.0.1:34254 AAPL,GOOGL,TSLA
```

Сервер отвечает:

```
OK
```

или при ошибке:

```
ERR <причина>
```

После ответа TCP-соединение закрывается.

### Стриминг данных (UDP)

Сервер отправляет котировки в формате JSON:

```json
{"ticker":"AAPL","price":183.42,"volume":3821,"timestamp":1713500000000}
```

### Keep-Alive (Ping/Pong)

Клиент каждые **2 секунды** отправляет строку `PING` по UDP на адрес сервера.

Если сервер не получает `PING` более **5 секунд** — прекращает стриминг для этого клиента.

## Уровни логирования

Управляются переменной окружения `RUST_LOG`:

| Команда | Что показывает |
|---|---|
| `RUST_LOG=error` | только критические ошибки |
| `RUST_LOG=warn` | ошибки и предупреждения |
| `RUST_LOG=info` | основные события (подключения, старт/стоп) |
| `RUST_LOG=debug` | всё включая каждый PING |

Пример:

```bash
RUST_LOG=debug cargo run -p quote_server
```

## Несколько клиентов

Сервер поддерживает несколько одновременных подключений.
Каждый клиент получает свой поток котировок с независимой фильтрацией по тикерам.

Запустить второй клиент на другом порту:

```bash
cargo run -p quote_client -- --udp-port 34255 --tickers-file tickers.txt
```

## Константы протокола

Находятся в `common/src/lib.rs`:

| Константа | Значение | Описание |
|---|---|---|
| `SERVER_TCP_PORT` | `7777` | TCP-порт сервера |
| `QUOTE_INTERVAL_MS` | `500` | Интервал генерации котировок (мс) |
| `PING_INTERVAL_SECS` | `2` | Интервал отправки PING клиентом (сек) |
| `PING_TIMEOUT_SECS` | `5` | Таймаут без PING до отключения (сек) |
| `STREAM_POLL_MS` | `100` | Интервал опроса канала в потоке стриминга (мс) |
